配置指南：
    1、CVI-100开头的规则是日志扫描配置:
        1)、将.xml中language语言后缀结束的、没有匹配到match规则的文件扫描出来。
        2)、当需要对检查多条规则时，可以自行添加规则。

    2、CVI-110开头的规则是检查匹配到内容的上面是否有写注释：
        1)、如果匹配到的函数、类上面没有写注释，则打印
        2)、只扫描-t/src中.xml中配置的language后缀的文件

    3、CVI-120开头的规则是检查C++中变量是否存在重复赋值：
            如c++中一个函数内存在如下情况视为变量重复赋值：
                int a() {
                    a= sResultSet->bbb();
                    a= sResultSetbbb->bbb();
                    return 0;
                    }
        1)、如果匹配到的函数内变量重复赋值，则打印
        2)、只扫描-t/src中.xml中配置的language后缀的文件

    4、CVI-200开头的文件为静态规则扫描：
        1)、如果-t文件中存在match匹配到的内容，视为缺陷
